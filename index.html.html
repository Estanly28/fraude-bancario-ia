// 01 · planteamiento

El problema del fraude bancario

El fraude en tarjetas de crédito genera pérdidas millonarias. Los sistemas basados en reglas estáticas fallan ante la sofisticación creciente de los ataques.

⚠️

DESBALANCE CRÍTICO

Solo 492 fraudes de 284,807 transacciones — el 0.17%. Un modelo que dijera siempre "legítima" tendría 99.83% de accuracy sin detectar absolutamente nada.

🔍

DETECCIÓN COMPLEJA

Las 28 variables del dataset son componentes PCA — el banco anonimizó los datos originales. El modelo debe aprender patrones abstractos sin saber qué representan.

🎯

OBJETIVO

AUC-ROC > 0.95 y F1-score > 0.90 sobre datos reales. Detección en tiempo real con bajo índice de falsos positivos para no bloquear clientes legítimos.

Dataset: Credit Card Fraud Detection — Kaggle · Universidad Libre de Bruselas · Transacciones europeas de septiembre 2013 · Variables anonimizadas con PCA · Único campo sin cifrar: Amount (monto en euros).

// 04 · resultados

Evolución del MLP — v1 a v4

Experimentación sistemática para encontrar la combinación óptima de técnicas. No siempre más complejo es mejor.

Versión Estrategia AUC-ROC F1-score Precision Recall
MLP v1 SMOTE únicamente 0.9818 0.8283 0.8200 0.8367
MLP v2 BatchNorm + 128→64→32→16 0.9769 0.7793 0.7217 0.8469
MLP v3 Class_weight sin SMOTE 0.9772 0.8163 0.8163 0.8163
MLP v4 ★ SMOTE + Class_weight + L2 0.9772 0.8410 0.8163 0.8469

Hallazgo clave: el umbral de decisión estándar de 0.50 producía F1 = 0.16 y precisión = 8.75%. El ajuste mediante curva Precision-Recall al umbral óptimo de 0.9962 elevó el F1 a 0.8283 sin cambiar la arquitectura. El umbral nunca debe asumirse como 0.50 en datos desbalanceados.

def construir_mlp_v4(input_dim):
    modelo = keras.Sequential([
        layers.Input(shape=(input_dim,)),

        # Capa 1: Dense + L2 + Dropout
        layers.Dense(64, activation='relu',
                     kernel_regularizer=regularizers.l2(0.0005)),
        layers.Dropout(0.3),

        # Capa 2
        layers.Dense(32, activation='relu',
                     kernel_regularizer=regularizers.l2(0.0005)),
        layers.Dropout(0.3),

        layers.Dense(16, activation='relu'),
        layers.Dense(1,  activation='sigmoid')
    ])
    modelo.compile(
        optimizer=keras.optimizers.Adam(learning_rate=0.001),
        loss='binary_crossentropy',
        metrics=['accuracy', AUC, Precision, Recall]
    )
    return modelo

Umbral óptimo via Precision-Recall

from sklearn.metrics import precision_recall_curve

precisions, recalls, umbrales = precision_recall_curve(y_test, y_pred_prob)
f1_scores = 2 * (precisions * recalls) / (precisions + recalls + 1e-8)

# Encontrar el umbral que maximiza F1
mejor_idx    = np.argmax(f1_scores)
mejor_umbral = umbrales[mejor_idx]

# Resultado: 0.50 → F1=0.16 · 0.9962 → F1=0.83
y_pred = (y_pred_prob >= mejor_umbral).astype(int)
